# 🤖 Parquet → Iceberg Migration Agent

AI-агент для автоматической миграции кода работы с данными из формата Apache Parquet в Apache Iceberg.

## 🚀 Почему агентный подход?

Миграция с Parquet на Iceberg — это не просто текстовая замена. Требуется:
- Контекстный анализ кода (PySpark, SQL, Pandas)
- Адаптация под разные паттерны партиционирования и схем
- Оценка влияния на data pipeline (батч → микро-батч)
- Принятие решений при ошибках и edge-cases

Жёсткие скрипты (regex/AST) ломаются на реальном коде. Агент с инструментами **сам решает порядок действий**, проверяет результат и даёт объяснимые рекомендации.

## 🛠 Архитектура и логика инструментов

| Инструмент | Назначение | Почему отдельно? |
|------------|------------|------------------|
| `analyze_code` | Сканирование файлов, поиск паттернов Parquet | Выделяет этап сбора данных. Позволяет LLM видеть полную картину до генерации. |
| `apply_iceberg_migration` | Применение правил миграции, генерация кода | Изолирует логику трансформации. Упрощает отладку и аудит. |
| `validate_and_assess` | Проверка результата + оценка data velocity | Финальный контроль + архитектурные рекомендации. Позволяет итеративно улучшать результат. |

**Почему не монолит?** Разделение позволяет LLM:
- Остановиться, если анализ не дал результатов
- Запросить уточнения перед миграцией
- Вернуться к анализу, если валидация нашла проблемы
